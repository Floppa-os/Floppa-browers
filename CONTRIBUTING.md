Данная инструкция предназначена для разработчиков, желающих внести свой вклад в развитие проекта. Пожалуйста, придерживайтесь описанных ниже стандартов и процедур, чтобы поддерживать качество и читаемость кодовой базы.

## Начало работы

Перед тем как приступить к написанию кода, убедитесь, что в вашей системе установлен Python версии 3.10 или выше. Основная библиотека для разработки интерфейса — PyQt6.

### Подготовка окружения

Для работы над проектом рекомендуется выполнить следующие шаги:

1. Создайте форк основного репозитория.
2. Клонируйте свой форк на локальный компьютер.
3. Настройте виртуальное окружение:
   \[ python -m venv venv \]
4. Активируйте окружение и установите необходимые зависимости.

## Процесс внесения изменений

Мы работаем по модели Pull Requests. Для успешного слияния вашего кода следуйте алгоритму:

1. Создайте тематическую ветку для новой функции или исправления ошибки:
   \[ git checkout -b feature/название-функции \]
   или
   \[ git checkout -b fix/описание-ошибки \]
2. Реализуйте ваши изменения, стараясь сохранять атомарность коммитов.
3. Оформите коммит с понятным описанием на русском языке.
4. Отправьте изменения в свой форк и создайте Pull Request в основной репозиторий.

## Стандарты написания кода

Соблюдение правил оформления кода позволяет всей команде быстрее понимать структуру проекта.

### Работа с графическим интерфейсом

Поскольку проект базируется на PyQt6, при создании виджетов следуйте принятой системе именования элементов. Группируйте элементы интерфейса в отдельные модули, если они представляют собой логически завершенные блоки.

## Создание баг-репортов и предложений

Если вы обнаружили ошибку или хотите предложить улучшение:

1. Ознакомьтесь с текущим списком задач в разделе Issues, чтобы избежать дублирования.
2. При описании ошибки укажите шаги для её воспроизведения, а также ожидаемый и фактический результат.
3. При предложении новой функции детально опишите проблему, которую она решает.
